1) Нужно сделать вебвью приложение - то есть человек открывает приложение и видит перед собой зашитую ссылку. Если интернета нет - видит сообщение типа подключите интернет.

2) Кроме того, в приложение нужно зашить SDK appsflyer.

К чему тут appsflyer - нужно почитать документацию, как это сделать: нам нужно передавать в сам урл вебвью приложения параметры из ссылки, через которую человек устанавливает это приложение из гугл плей.
Например, мы льем по ссылке с utm метками (utm_campaign, utm_source, utm_medium) с сайта №1, человек открывает приложение и видит перед собой в приложении сайт №1. Если человек установил приложение с сайта №2, то в вебвью он должен видеть другой урл.

Вот дока аппсфлаера - https://support.appsflyer.com/hc/en-us
Почитать что это вообще такое - https://lifehacker.ru/2015/09/07/obzor-instrumentov-analitiki-ios-prilozhenij-vse-chto-nuzhno-znat-dlya-uspeshnoj-raskrutki/
В гугл плей траф будет приходить в виде - https://play.google.com/store/apps/details?id=com.XXX&referrer=111_222_333
Где 111 - utm_campaign, 222 - utm_source, 333 - utm_medium


+  Ссылка, зашитая в приложение должна иметь вид - http://1.arrrrrr.net/?{utm_campaign}&s2={utm_source}&s3={utm_medium}

Если внутри приложения utm метки не передались (человек установил по ссылке без параметров, либо аппсфлаер не сумел передать их верно) - присваиваем им значение empty. То есть utm_medium=empty и т.д.

Те utm метки, по которым человек установил приложение сохраняем в данные приложения. То есть ссылка для пользователя должна быть всегда одной и той же.

Оформление приложения, иконка - на твой вкус по тематике онлайн тестов.

Название пакета - любое. Проект главное сохрани, потом пригодится.
